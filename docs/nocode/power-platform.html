# Use the SalesTim API with Power Platform beta

Abstract: This article explains how to register the SalesTim API as a a custom connector for the Microsoft Power Platform. The SalesTim Connector for Power Platform gives you access to all the powerful features of our Governance API, such as managing your teams or start a new team provisioning job, right from PowerApps, Power Automate.

⏱ Expected Duration

Creating the SalesTim Connector for Power Platform only requires a few operations that are fully described hereafter as a detailed step-by-step procedure. Assuming you have all the required access and permissions to execute it properly, it should take less than 10 minutes.

# Procedure Overview

# Create a new app registration in Azure Active Directory

To securely access your Microsoft 365 environment through the Microsoft Graph APIs, the first step is to create a dedicated app registration. An Azure AD app registration identifies a third-party app such as SalesTim, and defines the permissions you wan to grant to it. To learn more, you can refer to How and why applications are added to Azure AD.

To create a new app registration, follow these steps:

  1. Open your Azure Active Directory portal
  2. Select App registrations from the left menu
  3. Click New registration from the top bar
  4. Give the app a name, such as:
SalesTim Power Platform Connector
  1. Select the option Accounts in this organizational directory only, as you want to restrict access to your own tenant.
  2. Use this web redirection URL:
https://global.consent.azure-apim.net/redirect
  1. Click Register
  2. From the Overview menu, copy the Application (client) ID, and keep it as we're gonna reuse it later. You can temporarily paste it here:

Application (client) ID:

  1. Open the Certificates and secrets menu and click New client secret. A client secret is a kind of a password for your app, so manage it carefully.
  2. Give a name to your client secret such as:
SalesTim Power Platform Connector client secret
  1. Select the option Never to prevent this secret to expire in the future, and click Add
  2. Copy the Client secret value (be careful, it will only be shown once), and keep it as we're gonna reuse it later. You can temporarily paste it here:

Client secret:

  1. Open the API permissions from the left menu and click Add a permission from the top bar
  2. Select Microsoft Graph, then Delegated permissions
  3. From the permissions list, select:
    • OpenId permissions >
      • email
      • offline_access
      • openid
      • profile
    • Directory >
      • Directory.AccessAsUser.All
    • Group >
      • Group.ReadWrite.All
    • InformationProtectionPolicy >
      • InformationProtectionPolicy.Read
    • Mail >
      • Mail.Send
    • User >
      • User.Read
      • User.Read.All
  4. Click Add permissions
  5. Then click Grant admin consent for... then Yes

You're done, you've created your app registration for the SalesTim Connector. You should also have saved for later the Application (client) ID and Client secret that we're gonna use in the next steps.

# Import the SalesTim OpenAPI definition

Now that we've created the app registration, we're gonna use it to create our custom connector.

  1. To import the SalesTim API OpenAPI definitions for Power Automate and Power Apps, go to PowerApps or Power Automate.

TIP

The custom connector you're about to create will be available for both PowerApps and Power Automate, as they're sharing the same list of connectors.

  1. In the navigation pane, select Data then Custom connectors.
  • Select New custom connector then choose Import an OpenAPI from URL
  • Give your connector a name, such as :
SalesTim Connector
  1. Paste this URL:
https://dist.salestim.com/api/v1.0/open-api/power-platform/apiDefinition.swagger.json
  1. Click Import, then Continue
  2. Upload the connector logo that you can download from:
https://www.salestim.com/wp-content/uploads/2019/05/color.png
  1. Set the icon background color to:
#000000
  1. Click Security
  2. Ensure that the authentication type is set to OAuth 2.0 and the identity provider is set to Azure Active Directory
  3. Paste the previously copied Client id and Client secret
  4. Ensure that the login url is set to:
https://login.windows.net
  1. Ensure that the tenant ID is set to: common
  2. Set the resource URL to:
https://graph.microsoft.com
  1. Set the scope to:
https://graph.microsoft.com/.default`
  1. Click Create connector, then Close

# Next Steps

Now that you've created the SalesTim Connector for Power Platform, you can use it from both PowerApps and Power Automate.
